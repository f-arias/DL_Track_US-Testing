▶️Inicio: Interacción del Usuario

El proceso comienza cuando el usuario, en la interfaz gráfica principal de DL_Track_US, hace clic en el botón "Analyze Batch".

  • El método asociado a este botón es self.analyzeBatch().
↓

⚙️Preparación del Análisis: Método analyzeBatch()

Este método actúa como el centro de control inicial. No realiza el análisis directamente, sino que lo prepara y delega.

  • Recolecta todas las configuraciones de la GUI: rutas de archivos, tipo de imagen, parámetros de análisis (del `settings.json`), etc.
  • Inicia un nuevo hilo de ejecución (threading.Thread) para no congelar la interfaz gráfica.
  • El objetivo (target) de este hilo es la función calculateBatch(), a la cual le pasa todas las configuraciones como argumentos.
↓
🔄 Bucle Principal en calculateBatch(): Procesamiento por Imagen

🖼️Paso 1: Carga y Preprocesamiento de Imagen Individual

Para cada ruta de imagen encontrada, se llama a la función importAndReshapeImage().

  • Carga: cv2.imread(imagepath, 1) carga la imagen completa a color (3 canales).
  • Flip (Opcional): Si se indica, la imagen se invierte horizontalmente con np.fliplr().
  • Copia para Visualización: Se guarda una copia (img_copy) de la imagen antes del redimensionamiento, que será la base para la figura final.
  • Formato para el Modelo: La imagen se transforma a una forma 4D (1, 512, 512, 3) y se normaliza dividiendo por 255.0. Esta es la variable img que se pasará al modelo.

⚠️Punto Crítico: Ausencia de Recorte del Marco

El flujo estándar NO incluye una etapa de recorte. importAndReshapeImage redimensiona la imagen completa, incluyendo cualquier marco de información (datos del paciente, barras de escala, etc.). Esto implica que el usuario debe proporcionar imágenes ya limpias para obtener resultados óptimos.

↓

📏Paso 2 (Condicional): Calibración de Escala

Si el método de escalado seleccionado es "Barra", se llama a calibrateDistanceStatic().

  • Esta función analiza una porción del lado derecho de la imagen original sin redimensionar (nonflipped_img) para detectar las barras blancas de la escala.
  • Calcula la distancia en píxeles (calib_dist) que corresponde a una distancia conocida (e.g., 10 mm).
  • Importante: Esta función solo devuelve un valor numérico, no una imagen recortada.
↓

🧠Paso 3: Análisis Central - doCalculations()

Este es el corazón del algoritmo, donde ocurren la predicción y los cálculos geométricos.

  1. Predicción CNN:
    • Se ejecuta model_apo.predict(img) y model_fasc.predict(img).
    • Las máscaras de probabilidad resultantes se umbralizan para crear máscaras binarias.
  2. Refinamiento de Aponeurosis:
    • cv2.findContours sobre la máscara para obtener contornos crudos.
    • Filtrado por longitud y ordenamiento con sortContours().
    • Esqueletización (skeletonize) para obtener líneas de 1 píxel de grosor, seguido de operaciones morfológicas (dilate, erode) para limpiar y conectar.
    • Se extraen los contornos finales y limpios (contoursE).
  3. Extracción de Bordes y Modelado Matemático:
    • Se llama a contourEdge() sobre contoursE para obtener los puntos (x, y) de los bordes relevantes.
    • Se suavizan estos bordes con savgol_filter(), obteniendo upp_y_new y low_y_new.
    • Se ajusta un polinomio de grado 1 (np.polyfit) a estos bordes suavizados. Esto crea un modelo matemático (línea recta) para cada aponeurosis.
  4. Cálculo de Parámetros:
    • Se mide el grosor muscular (midthick) entre las aponeurosis suavizadas.
    • Se inicia un bucle para cada fascículo detectado. Para cada uno:
      • Se ajusta una línea recta.
      • Se calculan las intersecciones de esta línea con las líneas **extrapoladas** de las aponeurosis.
      • Se calcula la longitud del fascículo (FL) y el ángulo de pennación (PA).
  5. Generación de la Figura de Salida:
    • Se crea una figura de Matplotlib (fig).
    • Se dibuja la img_copy (la imagen de visualización).
    • Se dibujan las aponeurosis usando las coordenadas suavizadas (upp_x, upp_y_new) y (low_x, low_y_new).
    • Se dibujan los fascículos calculados.
↓

💾Paso 4: Agregación de Resultados de la Imagen Actual

  • Los resultados numéricos (listas de FL, PA, MT) se añaden a listas maestras (fascicles_all, etc.).
  • La figura fig generada por doCalculations() se guarda en un archivo PDF común (ResultImages.pdf).
↓

📊Paso 5: Post-Procesamiento Final

Después de que el bucle ha terminado de procesar todas las imágenes, se realizan operaciones finales sobre el conjunto completo de datos.

  • Se aplica un filtro Hampel (hampelFilterList) a las series de datos de FL y PA para eliminar valores atípicos.
↓

📄Paso 6: Exportación Final de Datos

Se llama a la función exportToExcel().

  • Organiza todos los datos brutos y filtrados en un DataFrame de Pandas.
  • Guarda el DataFrame en un único archivo .xlsx en la carpeta de resultados.
↓

🏁Fin del Proceso
